Introduction

Internet, Web et applications web sont des mots que nous entendons tous les jours mais qui sont pourtant sujets à de nombreuses confusions. Il semble primordial d’en donner une définition précise.

Internet est un réseau de réseaux à l’échelle mondiale. Il permet l’échange de paquets à travers ce réseau. Le Web n’est qu’une application d’Internet et permet de consulter des pages accessibles sur des sites à l’aide d’un navigateur. Enfin, les applications web sont des applications hébergées par des serveurs et accessibles par des navigateurs web. Ces applications sont nombreuses et variées : réseaux sociaux, webmail, sites de e-commerce, jeux. Il convient donc de sécuriser le Web qui contient des données sensibles, confidentielles ou personnelles.

De nombreux acteurs aident à se prémunir contre les attaques des hackers. En France, l’ANSSI qui s’occupe de la sécurité des systèmes d’information met à disposition des guides sur les bonnes pratiques pour la sécurité des systèmes informatiques. Aux Etats-Unis, l’OWASP est une communauté de bénévoles dédiée à la sécurité des applications web. Elle publie des articles, guides et logiciels pour sensibiliser aux risques liés à la sécurité web. De même, le WASC qui rassemble des experts en sécurité, publie des documents sur la sécurité web.

Architecture d’internet et fonctionnement du Web

Le Web repose sur le réseau internet, il convient donc d’en décrire brièvement le fonctionnement. Internet est construit en différentes couches. Le modèle majoritairement utilisé est le couple TCP/IP qui fonctionne en quatre couches : la couche accès réseau, la couche internet, la couche transport et la couche applicative.

Le Web utilise toutes ces couches pour ses échanges de données. Ce qui distingue les différents services d’Internet est le protocole utilisé par la couche applicative. Par exemple, pour échanger des mails le protocole SMTP est utilisé. Le Web fonctionne grâce au protocole HTTP.

Le protocole HTTP a été développé pour le Web et a permis son développement. C’est un protocole de messages : l’utilisateur envoie une requête par son navigateur web et la requête est traitée par le serveur qui renvoie une réponse au client.

Le Web a été créé en 1989 et a connu une évolution importante.

L’évolution du Web : du Web 1.0 au Web 3.0

Web 1.0 (1990-2000) : le Web statique

Dans les années 90, le Web avait pour vocation d’afficher uniquement du contenu aux utilisateurs, c’est pourquoi on parle de Web statique. Les sites web pouvaient être vus comme des magazines. Ils étaient limités à des liens hypertextes et à du contenu multimédia.

Web 2.0 (2000-2010) : le Web participatif axé sur les utilisateurs

A partir des années 2000, le comportement des utilisateurs a changé et le Web est devenu collaboratif. L’internaute ne consulte plus uniquement, il peut maintenant publier du contenu et est sollicité à le faire. Le Web 2.0 voit apparaître de multiples réseaux sociaux ou blog comme Facebook et YouTube.

Web 3.0 (2010-20xx) : le Web sémantique ou le Web de l’Internet des objets

Avec la multiplication des téléphones mobiles, tablettes et objets connectés, le Web a dû évoluer pour se connecter au monde réel des utilisateurs. Les maisons connectées sont un très bon exemple, le chauffage peut alors s’adapter à la météo. Ce Web est également centré sur l’utilisateur, ce n’est plus l’utilisateur qui agrège les données mais des suggestions viennent à lui basées sur son historique ou ses données personnelles. Cette évolution implique une évolution des technologies et de potentielles incompatibilités entre elles. Pour guider les différentes technologies du Web dans une même direction, le W3C pour World Wide Web Consortium est une communauté internationale qui s’occupe de développer les normes du Web.

Pourquoi sécuriser son site web ?

Le Web n’est plus uniquement utilisé pour afficher du contenu statique aux utilisateurs. Il présente différentes applications : moteurs de recherche, webmail qui permet de gérer ses mails par une interface web, le commerce électronique et les jeux en ligne, les réseaux sociaux. Toutes ces applications peuvent être sujettes à des actes de piratage.

Il n’est pas rare de voir dans les actualités des cas de piratage informatique. Par exemple en 2016, le site de e-commerce d’Acer a été victime d’une attaque de hackers qui ont réussi à dérober les informations personnelles de 34 500 clients incluant leurs coordonnées bancaires. En 2017, Anonymous a saturé le site de la Cour Constitutionnelle espagnole pour montrer leur soutien quant à l’indépendance de la Catalogne. Des attaques ont aussi été menées sur le site de la police espagnole. Ces exemples montrent que les attaques web sont utilisées pour différentes fins et ne sont pas rares. La sécurisation des sites web est donc un enjeu majeur.
